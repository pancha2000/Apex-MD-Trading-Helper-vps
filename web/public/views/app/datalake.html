📖 How It Works
🔷 Coins IN the Lake
  • Historical data stored in MongoDB
  • Gap auto-filled since last sync
  • Analysis uses stored + live data
  • Instant — no Binance API call needed
🔶 Coins NOT in the Lake
  • Candles fetched live from Binance
  • Stored in a temporary session (48h TTL)
  • Entry/TP/SL updates use the live session
  • Session auto-deleted when trade closes
⏰ Auto-Sync
  • Runs daily at 00:05 UTC
  • Only fetches MISSING candles
  • No duplicate data ever written
  • Safe to run anytime manually
